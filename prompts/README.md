## Контекст и очистка

### context_overflow_summary.txt
**Когда используется:** При переполнении контекста перед созданием протокола сессии.

**Вызывается в:** `_ollama_summarize_and_reset_context()`

**Переменные:**
- `{{REASON}}` - причина очистки (hard_ctx_threshold, repetition_detected и т.д.)
- `{{ORIGINAL_TASK}}` - последний запрос пользователя (обрезан 300 символов)

**Назначение:** Системный промпт для модели, чтобы сформировать краткий протокол сессии.

---

### context_overflow_user.txt
**Когда используется:** Пользовательский запрос к модели для создания протокола.

**Вызывается в:** `_ollama_summarize_and_reset_context()`

**Переменные:**
- `{{REASON}}` - причина очистки
- `{{ORIGINAL_TASK}}` - последний запрос пользователя (500 символов)
- `{{ARTIFACT_PATH}}` - путь к файлу с полной историей

**Назначение:** Конкретная задача для модели - создать протокол с обязательным указанием original_task.

---

### context_overflow_protocol.txt
**Когда используется:** После создания протокола, для продолжения работы с очищенным контекстом.

**Вызывается в:** `_ollama_summarize_and_reset_context()`

**Переменные:**
- `{{ARTIFACT_PATH}}` - путь к сохраненной полной истории
- `{{SESSION_PROTOCOL}}` - текст сгенерированного протокола
- `{{ORIGINAL_TASK}}` - задача пользователя (300 символов)

**Назначение:** Финальный системный промпт после очистки контекста - указывает модели продолжить выполнение задачи.

---

## Авто-продолжение

### auto_continue.txt
**Когда используется:** После очистки контекста для продолжения работы.

**Вызывается в:** `ask_ollama_stream()` - при max_tool_rounds, repetition_detected, hard_ctx_threshold

**Переменные:**
- `{{LAST_USER_PROMPT}}` - последний запрос пользователя
- `{{SESSION_PATH}}` - путь к сессии
- `{{ARTIFACT_PATH}}` - путь к артефакту с историей

**Назначение:** Просит модель продолжить выполнение задачи после очистки контекста, указывает где искать детали.

---

### auto_continue_final.txt
**Когда используется:** Когда модель сгенерировала thinking но не дала финальный ответ (missing_final_response).

**Вызывается в:** `ask_ollama_stream()`

**Переменные:**
- `{{LAST_USER_PROMPT}}` - последний запрос пользователя
- `{{SESSION_PATH}}` - путь к сессии

**Назначение:** Просит модель сформулировать финальный ответ без вызова инструментов.

---

## Прочие промпты

### chat_only.txt
**Когда используется:** Для моделей без поддержки tools (MODELS_NO_TOOLS).

**Вызывается в:** `_ensure_chat_only_system_message()`

**Переменные:** Нет

**Назначение:** Указывает модели что инструменты недоступны, просит отвечать текстом.

---

### context_trimmed.txt
**Когда используется:** При авто-сокращении контекста (не полная очистка, а обрезка старых сообщений).

**Вызывается в:** `_prepare_messages_for_ollama()`

**Переменные:**
- `{{ARTIFACT_PATH}}` - путь к файлу с обрезанными сообщениями

**Назначение:** Уведомление модели что часть истории была сохранена в артефакт.

---

### proofreader.txt
**Когда используется:** Для proofreader агента (режим корректуры).

**Вызывается в:** `proofreader_agent.py` (если есть)

**Переменные:** Зависят от реализации

**Назначение:** Инструкции для корректора текста.

---

## Устаревшие/Резервные

### broken_tools.txt
**Когда используется:** Если инструменты не загрузились.

**Назначение:** Уведомление о недоступных инструментах.

---

### dangerous_mode.txt
**Когда используется:** В dangerous mode (разрешены shell_exec, code_editor).

**Назначение:** Предупреждение об опасном режиме.

---

### resume_context.txt
**Когда используется:** При возобновлении сессии.

**Назначение:** Контекст для продолжения прерванной сессии.

---

### session_files.txt
**Когда используется:** Описание структуры сессии.

**Назначение:** Описание файловой структуры сессии для модели.

---

### session_location.txt
**Когда используется:** Указание пути к сессии.

**Назначение:** Информация о расположении сессии.

---

### system_time.txt
**Когда используется:** Добавление текущего времени.

**Назначение:** Текущее время для контекста.

---

### tool_policy.txt
**Когда используется:** Описание политики использования инструментов.

**Назначение:** Правила вызова инструментов.

---

### tool_reminder.txt
**Когда используется:** Напоминание о доступных инструментах.

**Назначение:** Напоминание модели о доступных инструментах.

---

## Как добавить новый промпт

1. Создать файл `.txt` в папке `prompts/`
2. Использовать переменные в формате `{{VAR_NAME}}`
3. Загружать в коде через `sm.load_prompt(session_path, "filename", VAR=value)`
4. Добавить описание в этот README

## Примечание

При старте сессии все промпты копируются из глобальной папки `prompts/` в папку сессии (`session_path/prompts/`), что позволяет кастомизировать промпты для конкретной сессии.

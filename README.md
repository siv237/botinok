# Проект «Ботинок» (Botinok)

Агентный ИИ с экстремальным фокусом на производительность и детальный аудит таймингов.

## Концепция
Цель проекта — создание консольного агента, способного эффективно работать на ограниченном железе (NVIDIA P104-100 8GB) за счет глубокой оптимизации рабочих процессов и использования специализированных моделей.

## Технический стек
- **Backend:** Ollama (развернута на `ollama.localnet`)
- **GPU:** P104-100 8GB VRAM
- **Базовая модель:** `qwen3.5:4b` (выбрана из-за 100% GPU offload и скорости)
- **Альтернатива:** `qwen3.5:9b` (требует >12GB RAM для стабильной загрузки, частично работает на CPU)
- **Контекст:** По умолчанию 8192 токена (для модели 4b).

## Архитектура сессий
Каждое взаимодействие — это отдельная сессия в директории `sessions/<timestamp>_<name>/`.
Внутри сессии:
- `context.json` — текущее состояние памяти.
- `steps/` — детализированные логи каждого шага (запрос, ответ, метаданные Ollama).
- `performance.log` — CSV/JSON с таймингами:
    - `ttft` (Time To First Token)
    - `tps` (Tokens Per Second)
    - `load_time` (время загрузки модели)
    - `total_time`

## BOTINOK AGENT

Быстрый и диагностируемый бот для работы с Ollama, ориентированный на прозрачность работы LLM и инструментов.

## Особенности
- **Инструменты (Function Calling)**: Поддержка вызова внешних функций (например, веб-поиск через `lynx`).
- **Прозрачность**: Визуализация процесса размышлений (thinking), работы инструментов и детальная статистика.
- **Логирование**: Полная прослеживаемость каждого шага в сессиях.

## Архитектура инструментов
Все инструменты регистрируются в `core/tool_manager.py` и вызываются через основной цикл в `botinok.py`.

### Веб-поиск (`tools/web_search.py`)
Использует консольный браузер `lynx` для получения текстового дампа результатов DuckDuckGo.
- **Без API**: Работает напрямую с HTML-версией поиска.
- **Диагностика**: В интерфейсе отображается запрос, анимация ожидания и объем полученных данных (KB).

## Структура сессий
Каждый запуск создает папку в `sessions/`:
- `context.json`: Полная история диалога, включая вызовы инструментов и скрытые размышления.
- `tools.log`: Детальный лог работы инструментов (время, аргументы, **полный сырой ответ** и размер).
- `thinking.md` / `response.md`: Раздельные логи размышлений и финальных ответов в формате Markdown.
- `session_raw.log`: Потоковый лог чанков с таймингами.

## Установка (Installation)

Установка выполняется одной командой (требуются права root):

**через curl:**
```bash
curl -sSL https://raw.githubusercontent.com/siv237/botinok/main/install.sh | sudo bash
```

**через wget (если нет curl):**
```bash
wget -qO- https://raw.githubusercontent.com/siv237/botinok/main/install.sh | sudo bash
```

Скрипт автоматически:
1. Установит системные зависимости (`python3-venv`, `lynx`, `git`).
2. Развернет проект в `/opt/botinok`.
3. Создаст виртуальное окружение и установит библиотеки.
4. Добавит команду `botinok` в систему.

## Использование (после установки)

```bash
botinok
```

Дополнительные параметры:
- `-m MODEL`: Выбор модели (например, `qwen3.5:4b`).
- `-c CTX`: Размер контекста (по умолчанию `8192`).
- `-p "PROMPT"`: Начальный запрос (агент выполнит его и перейдет в интерактив).

## Настройка

Файл конфигурации находится в `/opt/botinok/config.cfg` (после установки) или в корне проекта. Вы можете изменить URL Ollama, модель по умолчанию и параметры инструментов.

## Требования
- Ollama (запущенная и доступная по сети).
- Python 3.10+.
- Lynx (устанавливается автоматически скриптом).

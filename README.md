# Проект «Ботинок» — Консольный Агентный ИИ

Агентный ИИ, ориентированный на работу с **Ollama**. Экстремальный фокус на производительность, прозрачность работы LLM и детальный аудит сессий. Полностью консольный интерфейс с богатым форматированием.

## Концепция
Цель проекта — создание автономного консольного агента, способного эффективно решать задачи программирования, поиска информации и системного администрирования в системах с малым объемом VRAM за счет глубокой оптимизации рабочих процессов.

## Основные возможности
- **Продвинутый консольный UI (Rich)**: Интерактивный интерфейс с цветовым форматированием, живой статистикой VRAM, TTFT (Time To First Token), TPS (Tokens Per Second) и визуализацией процесса размышлений (thinking).
- **Умное управление контекстом**: 
    - Автоматическая обрезка (trimming) истории для предотвращения переполнения.
    - Генерация **SESSION_PROTOCOL** при достижении лимитов: модель сама суммаризирует важные факты и намечает следующие шаги.
- **Система инструментов (Function Calling)**: Богатый набор инструментов для взаимодействия с миром.
- **Безопасность (Dangerous Mode)**: Потенциально опасные действия (правка кода, выполнение shell) требуют явного включения через переменную окружения.
- **Внешние системные промпты**: Все системные сообщения вынесены в `prompts/`. Каждая сессия получает свою копию, которую можно редактировать для тонкой настройки поведения агента.
- **Полная прослеживаемость**: Каждая сессия сохраняет все мысли, вызовы инструментов и артефакты в структурированном виде.

## Архитектура инструментов

Все инструменты динамически загружаются через `core/tool_manager.py`.

### 🌐 Сеть и поиск
- **web_search** — Поиск в интернете через DuckDuckGo (использует `lynx`)
- **open_url** — Извлечение текстового контента со страницы (lynx)
- **web_extract** — Извлечение структурированных данных (ссылки, изображения, таблицы) с HTML-страниц
- **web_extractor** — Алиас на web_extract для совместимости
- **curl** — HTTP GET запросы, скачивание файлов

### 🛠 Системные инструменты
- **file_system** — Навигация, поиск (grep, find), чтение и инспекция файлов. Продвинутый `find` с фильтрами по размеру, времени, типу.
- **shell_exec** — Выполнение bash-команд (требует `--dangerous`)
- **journal** — Анализ системных логов `journalctl`

### 💻 Разработка и Навыки (Skills)
- **code_editor** — Полноценное редактирование файлов (чтение, запись, замена фрагментов)
- **github** — Работа с репозиториями (clone, log, status, diff)
- **skills** — Мощная система расширения функционала
    - `Personal`: `~/.botinok/skills/` — приватные навыки пользователя
    - `Project`: `./skills/` — общие навыки проекта

### 🧠 Опыт и память
- **experience** — Локальная база данных «позитивного» и «негативного» опыта. Агент учится на своих ошибках

### 🖼️ Мультимодальность
- **vision** — Анализ изображений мультимодальной моделью
  - Автоматическая проверка формата и разрешения
  - Поддержка: JPEG, PNG, GIF, BMP, WebP (и любые другие через конвертацию)
  - Автоконвертация в RGB JPEG при необходимости (прозрачность, неподдерживаемые форматы)
  - Авторесайз до лимитов модели (4096×4096 для Qwen3.5, уменьшает но не увеличивает)
  - Источники: локальный файл или URL

**Примеры использования:**
```
Анализ фото: vision(image_path="/path/to/photo.jpg", prompt="Что на изображении?")
Скриншот сайта: vision(url="https://example.com/screenshot.png")
```

## Управление сессиями и контекстом

Каждый запуск создает директорию `sessions/<timestamp>_<name>/`.

### Структура сессии:
- `context.json`: Полная история диалога и состояние памяти.
- `thinking.md`: Лог всех размышлений модели (thinking block).
- `response.md`: Финальные ответы без служебной информации.
- `tools.log`: Детальный технический лог вызовов (время, аргументы, размер ответа).
- `artifacts/`: Сюда сохраняются дампы больших ответов инструментов (HTML, JSON, ошибки), чтобы не раздувать контекст LLM.
- `prompts/`: Копия системных промптов, специфичная для сессии.

## Системные промпты

Все системные сообщения вынесены в отдельные текстовые файлы в папке `prompts/`. При создании сессии они копируются в `session/prompts/`, что позволяет:
- Настраивать поведение агента для конкретной сессии
- Возвращаться к промптам сессии при продолжении работы
- Использовать глобальные промпты для новых сессий

Файлы промптов:
- `system_time.txt` — информация о текущем времени
- `session_location.txt` — пути к сессии и project/
- `session_files.txt` — карта файлов сессии
- `tool_policy.txt` — политика использования инструментов
- `tool_reminder.txt` — напоминание перед каждым ходом
- `dangerous_mode.txt` — статус dangerous mode
- `broken_tools.txt` — информация о неработающих инструментах
- `resume_context.txt` — контекст для продолжения сессии
- `chat_only.txt` — режим без инструментов
- `context_trimmed.txt` — уведомление при обрезке контекста
- `proofreader.txt` — инструкции для корректора

**Важно:** Если файл отсутствует в `session/prompts/`, используется версия из глобальной `prompts/` (или `/opt/botinok/prompts/` для установленной версии).

## Безопасность (Dangerous Mode)

По умолчанию опасные инструменты (`code_editor`, `shell_exec`) отключены. Чтобы разрешить их использование в текущей сессии, необходимо запустить агент с аргументом `--dangerous`:

```bash
botinok --dangerous
```
При использовании `shell_exec` агент всегда будет запрашивать подтверждение пользователя перед выполнением команды.

## Технический стек и Ollama

### Работа с Ollama (основной режим)
Ботинок спроектирован для работы с локальным или сетевым инстансом [Ollama](https://ollama.com).

**ВАЖНО:** Для полноценной работы инструментов (tools) необходимо использовать модели, поддерживающие **Function Calling** (например, `qwen2.5`, `llama3.1`, `mistral-nemo`).

**Установка Ollama:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Загрузка моделей:**
```bash
ollama pull qwen2.5:7b
ollama pull qwen3.5:4b
```

### Удаленные модели (Cloud/Remote)
Если локальных ресурсов недостаточно, Ollama позволяет использовать удаленные или облачные модели (например, `minimax-m2.7:cloud`). Для бота это выглядит как обычная модель Ollama, никакие дополнительные ключи или URL в самом боте настраивать не нужно — достаточно указать имя модели.

- **Backend:** Ollama (по умолчанию `localhost:11434`).
- **Базовая модель:** `qwen3.5:4b` или любая другая модель, доступная в вашем инстансе Ollama.
- **Контекст:** По умолчанию 8192 токена.

## Установка

**через curl:**
```bash
curl -sSL https://raw.githubusercontent.com/siv237/botinok/main/install.sh | sudo bash
```

Скрипт установит системные зависимости (`python3-venv`, `lynx`, `git`), развернет проект в `/opt/botinok` и создаст алиас `botinok`.

## Использование

### 1. Интерактивный режим (стандартный)
Запуск полноценного интерфейса с диагностикой:
```bash
botinok
```

### 2. Одиночный запрос
Агент выполнит задачу и предложит продолжить в интерактивном режиме:
```bash
botinok "Проанализируй логи в /var/log/syslog и найди ошибки сети"
```

### 3. Режим для скриптов (Stealth)
Выводит только финальный ответ модели без ASCII-арта и панелей статистики. Идеально для автоматизации:
```bash
botinok --stealth "Сколько оперативной памяти свободно?"
```

### 4. Работа в конвейере (Pipe mode)
Бот автоматически определяет наличие данных в `stdin`. Это позволяет использовать его для обработки вывода других команд:
```bash
# Передача логов для анализа
tail -n 100 /var/log/syslog | botinok "Найди критические ошибки"

# Передача содержимого файла
cat script.py | botinok "Объясни, что делает этот код"

# Быстрый вопрос через echo
echo "Как сварить кофе?" | botinok
```
В Pipe-режиме (если это не интерактивный терминал) бот автоматически переходит в `stealth` режим (выводит только ответ).

### 5. Выбор модели и контекста
```bash
botinok -m qwen3.5:9b -c 16384 --dangerous
```
Параметры:
- `-m MODEL`: Имя модели Ollama (по умолчанию из конфига).
- `-c CTX`: Размер контекста (например, `16384` или `32768`).
- `--dangerous`: Разовая активация опасных инструментов (`code_editor`, `shell_exec`) для текущей сессии.
- `--wizard`: Запуск интерактивного мастера настройки.
- `--update`: Проверить и установить обновления из git.
- `--debug`: Включить отладочный вывод.
- `--proofread`: Режим корректора (цикл: Исполнитель → Корректор).

### 6. Обновление
Проверить и установить обновления из git-репозитория:
```bash
botinok --update
```
Если доступна новая версия, будет показано сравнение версий и запрошено подтверждение перед установкой.

## Настройка
Вы можете быстро настроить URL Ollama и модели по умолчанию через мастер:
```bash
botinok --wizard
```
Конфигурационный файл сохраняется в `/opt/botinok/config.cfg` или по пути в переменной `BOTINOK_CONFIG`.

## Разработка новых навыков (Skills)
Новые навыки добавляются в папку `skills/`. Каждый навык должен содержать файл `SKILL.md` с описанием возможностей и реализацию на Python, которая подключается через `tools/skills.py`.
